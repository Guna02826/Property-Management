# Neorem - Property Hierarchy Structure & Functions

## 1. Portfolio

**Purpose:** Logical grouping of multiple properties for management and analysis

**Ownership:** Belongs to a single owner (typically) or organized by management company

### Core Functions

- Create, update, delete portfolio
- Assign/unassign properties to portfolio
- Assign Portfolio Manager(s) to portfolio
- Set portfolio-level settings and preferences
- Archive/reactivate portfolio

### Data Management Operations

- CRUD operations with audit logging
- Bulk property assignment/unassignment
- Portfolio cloning/duplication
- Data import/export for portfolio properties

### Financial Operations

- Aggregate financial metrics (total revenue, expenses, NOI)
- Portfolio-wide rent roll aggregation
- Comparative financial analysis across properties
- Portfolio-level budget vs. actuals
- Cash flow consolidation
- ROI and cap rate calculations

### Reporting & Analytics

- Portfolio dashboard with KPIs (occupancy, collection rate, NOI)
- Performance comparison reports (property-to-property, period-to-period)
- Financial summary reports
- Occupancy trend analysis
- Revenue forecasting
- Export portfolio reports (PDF, Excel, CSV)

### Access Control

- Portfolio-level user assignments
- Role-based access restrictions
- Data isolation per organization

## 2. Property

**Purpose:** Physical real estate asset (can be single building or complex)

**Types:** Supports both residential (housing) and commercial properties, as well as mixed-use developments

**Structure:** Contains one or more buildings (or standalone units for single-family)

### Core Functions

- Create, update, delete property
- Assign property to portfolio(s)
- Assign Property Manager(s) to property
- Manage property ownership and legal entity
- Track property status (active, inactive, under renovation, for sale, sold)
- Geocode and validate address
- Bulk unit creation/import

### Data Management Operations

- CRUD operations with version history
- Property information updates with audit trail
- Document management (deeds, permits, certificates, insurance)
- Photo and media management
- Property history tracking
- Bulk data import/export

### Financial Operations

- Property-level financial tracking (revenue, expenses, NOI)
- Rent roll aggregation for all units
- Expense tracking and categorization
- Budget vs. actuals reporting
- Cash flow tracking
- Property valuation and market analysis
- Tax and insurance tracking
- Depreciation calculations

### Reporting & Analytics

- Property dashboard with KPIs
- Occupancy and vacancy reports
- Financial performance reports
- Tenant turnover analysis
- Maintenance cost analysis
- Comparative analysis with portfolio averages
- Export property reports

### Compliance & Legal

- Compliance certificate tracking
- Inspection history
- Permit management
- Insurance policy tracking
- Regulatory compliance monitoring

### Access Control

- Property-level user assignments
- Property Manager assignments
- Role-based access restrictions

## 3. Building (Optional - for complexes)

**Purpose:** Physical structure within a property (for multi-building properties)

**Structure:** Contains multiple floors and units

**Note:** Optional level - not required for single-building or single-family properties

### Core Functions

- Create, update, delete building
- Assign building to property
- Manage building-specific information
- Track building status
- Manage building-level infrastructure

### Data Management Operations

- CRUD operations
- Building document management
- Floor plan management
- Infrastructure tracking (elevators, HVAC, etc.)

### Financial Operations

- Building-level expense allocation
- Building depreciation tracking
- Building insurance tracking

### Reporting & Analytics

- Building occupancy reports
- Building maintenance cost reports
- Unit performance by building

### Compliance & Legal

- Building permits and certificates
- Building inspections
- Elevator compliance tracking
- Fire safety compliance

## 4. Floor (Optional - for multi-story buildings)

**Purpose:** Level within a building (Ground, 1st, 2nd, Basement, etc.)

**Structure:** Contains multiple units, common spaces, and facilities

**Note:** Optional level - not required for single-story properties

### Core Functions

- Create, update, delete floor
- Assign floor to building
- Manage floor layout and space allocation
- Track floor-specific amenities
- Manage floor access control

### Data Management Operations

- CRUD operations
- Floor plan documentation
- Common space allocation
- Unit assignment to floor

### Reporting & Analytics

- Floor occupancy reports
- Floor-level unit performance
- Common space utilization

## 5. Unit / Space

**Purpose:** Leasable or usable space within property/building/floor

**Types:** Residential Unit, Commercial Unit, Common Space, Amenity Space, Utility Space

### Core Functions

- Create, update, delete unit
- Assign unit to property/building/floor
- Manage unit details (type, size, bedrooms, bathrooms, amenities)
- Track unit status (available, occupied, reserved, maintenance, unavailable)
- Assign parking spaces and storage units
- Manage unit photos and floor plans

### Data Management Operations

- CRUD operations with audit trail
- Unit document management
- Unit photo/video management
- Unit condition tracking
- Inspection history
- Move-in/move-out documentation

### Financial Operations

- Rent amount management
- Market rent tracking
- Unit-level revenue tracking
- Unit-specific expense allocation
- Security deposit tracking
- Rent increase history

### Lease Management

- Current lease tracking
- Lease history
- Lease renewal management
- Tenant assignment

### Reporting & Analytics

- Unit availability reports
- Unit performance metrics
- Rent roll by unit
- Unit maintenance history
- Unit profitability analysis

### Access Control

- Unit-level access restrictions
- Tenant access to own unit only

## 6. Sub-Spaces / Components (Optional)

**Purpose:** Detailed breakdown within units (rooms, parking, storage, balconies)

**Note:** Optional level - for advanced space management and detailed tracking

### Core Functions

- Create, update, delete sub-spaces
- Assign sub-spaces to units
- Track room-level details
- Manage parking space assignments
- Manage storage unit assignments

### Data Management Operations

- CRUD operations
- Space condition tracking
- Space documentation

### Financial Operations

- Parking fee tracking
- Storage fee tracking
- Space-specific cost allocation

## Supporting Elements (Cross-Level)

### 7. Common Spaces

- Shared areas within property/building/floor (lobbies, hallways, corridors, stairwells)
- Tracks location, square footage, maintenance responsibility, and usage restrictions
- Manages access control, cleaning schedules, and maintenance costs
- Supports common area factor calculations for commercial leases
- Can exist at property, building, or floor level

### 8. Amenities

- Recreational, fitness, social, or service facilities
- Types: Property-level (pool, gym, clubhouse), Building-level (lobby, rooftop), Unit-level (in-unit features)
- Tracks access levels, usage rules, operating hours, and capacity
- Manages booking requirements, fees, and maintenance schedules
- Supports amenity-specific financial tracking and insurance

### 9. Infrastructure Elements

- Physical systems and equipment (lifts/elevators, stairs, parking, utilities)
- Lifts/Elevators: Tracked at building/floor level with maintenance schedules and compliance
- Parking: Can be property-level (parking lot) or unit-level (assigned spaces)
- Utilities: Meters, HVAC systems, electrical panels tracked separately
- Manages maintenance schedules, service contracts, and compliance certificates

### 10. Parking Spaces

- Individual parking spots (covered, uncovered, garage, street)
- Tracks assignment to units/tenants, fees, and accessibility features
- Manages parking space dimensions, type, and EV charging availability
- Supports parking fee collection and space reservation
- Can be standalone or assigned to specific units

## Hierarchy Flow

```
Portfolio → Property → Building (optional) → Floor (optional) → Unit → Sub-Spaces (optional)
```

Supporting elements (Common Spaces, Amenities, Infrastructure, Parking) can exist at multiple levels and are linked to the appropriate parent level (Property, Building, Floor, or Unit)

## Cross-Level Operations

### Data Aggregation

- Roll-up financial data from Unit → Property → Portfolio
- Aggregate occupancy metrics across hierarchy levels
- Consolidate maintenance costs across levels
- Aggregate tenant data across properties

### Bulk Operations

- Bulk property creation/import
- Bulk unit creation/import
- Bulk status updates
- Bulk assignment operations
- Bulk data export

### Search & Filtering

- Multi-level search (across portfolios, properties, units)
- Advanced filtering by multiple criteria
- Hierarchical navigation
- Quick access to related entities

### Integration Points

- API endpoints for each hierarchy level
- Webhook events for hierarchy changes
- Data synchronization across levels
- External system integration points

### Performance Optimization

- Denormalized fields for quick access (property_id in unit, portfolio_id in property)
- Cached aggregations for dashboards
- Indexed queries for fast lookups
- Lazy loading for optional levels
